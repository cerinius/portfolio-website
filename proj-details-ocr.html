Optical Character Reader

At the start of my time at Niagara College (January 2018) I began working on this personal project. I have always had an interest in the technology of Optical Character Recognition (OCR). Early implementations of Google Lens always fascinated me, or scanning a cheque with my phone and it perfectly reading the information on it. From this interest, I decided to start working on my own desktop application that will take advantage of OCR. In my own time, I spend quite a bit of time playing video games. Often, games will come out in other regions of the world that interest me, and usually I would have to wait awhile for them to come to North America, or I can bare through trying to read a different language. To remedy this problem, the application that I wanted to create was one that allowed you to screengrab a region of your screen on your desktop, and pull all text from it that isn't english. From here, you can pick your desired language to translate to and it will run it through Google Translate. Due to the nature of a computer screen, drawn graphics like images, videos, or a game screen can't be simply copy/pasted. This tech makes it significantly easier to translate things that don't use roman characters like English.

This project was built using a Java client that allows users to drag a box around what region of the screen they'd like to capture, and it uses a Python 3 REST API for running translated text through Tesseract-OCR (An open-source OCR library). The REST API was created using Flask as a framework. In the GitHub repository, I have the open-source version of the client, and the original local Java server I designed before moving onto a Python API for performance issues. Local allows you to contain all of your images privately, while the client can be run alone to connect to my publicly hosted API.

See the entire project at my GitHub Repository and read through the more detailed walkthrough/documentation.
